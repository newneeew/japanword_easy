# 버그 수정 보고서 v2

## 수정일: 2026-03-23

## 문제 1: 화면 어두워짐 및 전환 지연
- **증상**: 단어 클릭 시 화면이 어두워지고, 다음 문제 전환 시 일시적 지연 발생
- **원인**: `time.sleep()`이 서버 사이드에서 스크립트 실행을 블로킹하여 Streamlit "Running..." 오버레이가 표시됨. sleep이 렌더링 전에 실행되어 사용자가 피드백을 볼 수 없음
- **해결**: `time.sleep()` 완전 제거. 클라이언트 사이드 JavaScript `setTimeout()`으로 대체. 숨겨진 버튼을 JS가 지연 후 클릭하여 서버에 rerun을 트리거. 정답 1.3초, 오답 1.7초 후 자동 전환. 서버 블로킹 없이 피드백이 즉시 표시됨

## 문제 2: 정답/오답 시각적 피드백 부재
- **증상**: 선택지를 골라도 어떤 것이 정답인지, 오답인지 시각적 구분이 불가능
- **원인**: 잠금 상태에서 선택지가 일반 버튼으로만 표시되어 색상 구분 없음
- **해결**: 잠금 상태에서 버튼 대신 CSS 스타일 카드로 교체. 정답=초록(✅, #dcfce7 배경, #22c55e 테두리), 오답=빨강(❌, #fee2e2 배경, #ef4444 테두리), 기타=회색(#f1f5f9). 각 카드 아래 🔊 발음 버튼으로 개별 단어 클릭 재생 유지

## 문제 3: 소리 미재생
- **증상**: 정답/오답 선택 후 일본어 음성이 재생되지 않음
- **원인**: `components.html`의 `<audio autoplay>` HTML 요소가 iframe 내에서 브라우저 autoplay 정책에 의해 차단됨
- **해결**: HTML `<audio>` 태그 대신 JavaScript `new Audio()` API 사용. JS Audio 객체의 `.play()` 호출은 사용자 인터랙션(버튼 클릭) 컨텍스트를 상속하여 autoplay 정책을 우회. gTTS 캐싱(`@st.cache_data`, max_entries=200)으로 동일 단어 재호출 시 네트워크 지연 없음

## 기술적 변경 요약
| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 자동 전환 | `time.sleep()` 서버 블로킹 | JS `setTimeout` + 숨겨진 버튼 클릭 |
| 선택지 피드백 | 이모지 prefix 버튼 | CSS 색상 카드 (초록/빨강/회색) |
| 음성 재생 | `<audio autoplay>` HTML | `new Audio().play()` JS API |
| import | `import time` 포함 | `time` 제거 |
